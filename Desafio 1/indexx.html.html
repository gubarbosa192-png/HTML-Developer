    • História do HTML

    Voltar

    • O HTML (HyperText Markup Language) foi criado por Tim Berners-Lee nos anos 90 com a finalidade inicial de tornar possivel o acesso e a troca de informações de documentação de pesquisas, entre cientistas de diferentes universidades.

    • Em 1995, a importância do HTML cresceu significativamente com a criação do Wide Web Consortium (W3C) , uma organização internacional que desenvolve padrões para a web. Nesse mesmo ano, foi lançada a versão 2.0 do HTML, que formalizou as características da linguagem e garantiu uma base sólida para seu desenvolvimento futuro.

    • Desde então, o HTML passou por várias atualizações, com a versão mais recente sendo o HTML5, lançado em 2014. O HTML5 trouxe uma série de melhorias e novos recursos, como suporte nativo para multimídia, gráficos vetoriais e APIs avançadas para desenvolvimento web. Hoje, o HTML é a espinha dorsal da web, permitindo a criação de páginas e aplicações interativas que são acessíveis a bilhões de pessoas em todo o mundo.

    Clients

    Voltar

    Client (Requisitante) é o dispositivo ou aplicação que inicia a comunicação fazendo uma solicitação. Exemplos comuns de clientes incluem navegadores, (Chrome, Firefox), aplicativos de celular (WhatsApp, Instagram) e computadores pessoais. O cliente foca na interação com o usuário.

    Servers

    Voltar

    Servidores (Provedor) O servidor processa essas solicitações, recupera as informações relevantes e as envia de volta ao cliente. Por exemplo, um servidor web responde com dados do site quando um navegador os solicita.

    Tipos comuns de servidores

    Existem diversos tipos de servidores, cada um especializado em uma função específica.

    • Servidor Web: Entrega páginas de sites (HTML, imagens) para o seu navegador. Exemplos: Apache e Nginx.
    • Servidor de Banco de Dados: Onde as informações organizadas ficam guardadas. Quando você faz login, é ele quem valida sua senha. Exemplos: MySQL e PostgreSQL.
    • Servidor de E-mail: Gerencia o envio e recebimento de mensagens através de protocolos como SMTP e IMAP.
    • Servidor de Arquivos: Centraliza o armazenamento de documentos para que várias pessoas em uma rede possam acessá-los com segurança.
    • Servidor DNS: Funciona como a "lista telefônica" da internet, convertendo nomes de domínios (como google.com) em endereços IP.

    Estrutura básica de um HTML

    Voltar

    A página inicial do site é, por uma convenção, chamada de index.html. Por padrão, diversos sites espalhados pela internet estarão utilizando essa nomenclatura para sua página principal.

    HTML é uma linguagem de marcação usada para criar páginas da web. Ela é composta por uma série de elementos, conhecidos como tags, que definem a estrutura e o conteúdo de um documento. A estrutura básica de um documento HTML inclui as seguintes tags principais:

    Um site é formado, de basicamente, por três tags principais:

    1. html: marca aquele pedaço de texto como HTML
    2. head: delimita a "cabeça" do site, o conteúdo que não aparece no corpo da página
    3. body: define aquele pedaço como o corpo da página, que conterá títulos, textos, fotos e mais

    Essas três tags HTML iniciais definem o básico do corpo de uma página HTML, dividindo a parte das configurações da página e a parte que o usuário vê (o corpo da página).

    Tags e suas funções

    Tags são os elementos básicos do HTML que definem a estrutura e o conteúdo de uma página web. Elas são usadas para criar títulos, parágrafos, listas, links, imagens e muito mais. Cada tag tem uma função específica e é delimitada por colchetes angulares (< >).

    Algumas das tags mais usadas no HTML são:

    • html: Define o início do documento HTML.
    • head: Contém metadados e links para scripts e estilos.
    • body: Contém todo o conteúdo visível da página.
    • h1 a h6: Usadas para títulos e subtítulos.
    • p: Define um parágrafo.
    • title: Define o título da página.
    • body: Contém o conteúdo da página.
    • h1 a h6: Títulos e subtítulos.
    • p: Parágrafos.
    • a: Links.
    • img: Imagens.
    • ul e ol: Listas não ordenadas e ordenadas.
    • li: Itens de lista.
    • div: Divisões ou seções de conteúdo.
    • del: Texto excluído.
    • i: Texto em itálico.

    Tags adicionais importantes:

    • link: Usada para vincular folhas de estilo externas.
    • meta: Fornece metadados sobre o documento HTML.
    • span: Usada para agrupar elementos em linha para aplicar estilos.
    • table: Cria tabelas.
    • form: Cria formulários para entrada de dados.

    Diferença entre listas ordenadas e não ordenadas

    Voltar

    Listas são formadas com uma tag mãe, definindo o tipo da lista, e os seus filhos, que seriam os itens dessa lista. As listas podem ser ordenadas < ol >< /ol > ou não ordenadas < ul >< /ul >. Haverão itens dentro dessas listas, determinadas pela tag < li >< /li >.

    Lista ordenada

    Listas ordenadas são construídas a partir da tag < ol >< /ol >, e dentro dessa tag, os itens com a tag < li >< /li >.
    1. Item 1
    2. Item 2
    3. Item 3

    Lista não ordenada

    Listas não ordenadas são construídas a partir da tag < ul >< /ul >, e dentro dessa tag, os itens com a tag < li >< /li >.
    • Item 1
    • Item 2
    • Item 3
